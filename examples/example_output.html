硬件类课程模板：图文绑定讲解

硬件类课程的核心不是把 slide 内容逐页复述，而是把 structure、signal flow 和 timing behavior 讲清楚。关键图片应放在对应主题下面，图后立即解释组成件、接口和信号方向。

知识地图

Memory map 决定 CPU 能访问哪些 peripheral registers；bit mask operations 决定如何修改寄存器中的单个字段；这些配置最终影响 GPIO / timer / interrupt 的硬件行为。

关系链：Register address -> Bit fields -> Peripheral configuration -> External hardware behavior。

核心概念速查

概念 / Term解释
GPIOGeneral-Purpose Input/Output，连接外部 pins 的通用输入输出端口。
Control register用于配置 peripheral 行为的 register，例如方向、模式、使能位。
Interrupt flag硬件事件发生后置位的状态位，software 需要读取并清除。

Memory-Mapped I/O（内存映射输入输出）

理解这部分时，应该先看图中的 CPU、bus、address decoder 和 peripheral register。CPU 发出 address 和 read/write control signals，address decoder 判断目标是否属于某个 peripheral，然后对应 register 被读取或写入。

这张图的关键不是“有几个方框”，而是 data path 和 control path 如何分工：data bus 传递 register value，control signals 决定这次访问是 read 还是 write。对于嵌入式程序来说，一行 C code 可能最终变成一次 bus transaction。

\[ \text{masked\_value} = \text{register\_value} \,\&\, \text{mask} \]

这里 \(\text{register\_value}\) 是当前 register 内容，\(\text{mask}\) 选择要观察的 bit field。这个公式常用于检查 hardware flag 是否置位。

AI / 算法类课程模板：方法主线 + 理论证明分离

AI、reinforcement learning 和算法课程通常需要先讲清楚 problem setting，再讲 objective、algorithm procedure 和理论依据。长证明不应打断主线，可以单独放在“理论证明 / 推导”部分。

知识地图

MDP 给出 state、action、reward 和 transition；value function 衡量长期收益；Bellman equation 把当前决策和未来收益连接起来；最后得到 policy improvement 或 learning update。

核心概念速查

概念 / Term解释
State智能体观察到的环境状态，记为 s。
Policy从 state 到 action 的决策规则，常写作 π(a|s)。
Value function从某个 state 或 state-action pair 出发的 expected return。

方法主线

以 policy evaluation 为例，目标不是直接找最优 action，而是在给定 policy π 时估计每个 state 的长期价值。直觉上，一个 state 的价值等于 immediate reward 加上下一步 state 的 discounted value。

\[ V^\pi(s)=\mathbb{E}_\pi\left[R_{t+1}+\gamma V^\pi(S_{t+1})\mid S_t=s\right] \]

其中 \(\gamma\) 是 discount factor，用来控制未来 reward 的权重；当 \(\gamma\) 越接近 1，模型越重视长期收益。

理论证明 / 推导：Bellman expectation equation

推导时先把 return 写成递归形式：当前 reward 加上折扣后的未来 return。再对 policy 和 transition probability 取 expectation，就得到 Bellman expectation equation。这个推导适合单独放在证明区，因为它支撑方法正确性，但不应该打断算法主线。

数学类课程模板：定义、定理、证明与例子

数学课程应优先转写并解释 definitions、theorems、proof ideas 和 examples。除非 slides 中的图是几何直觉或 visual proof 的关键，否则不需要放大量截图。

知识地图

Vector space 提供对象集合；linear independence 描述向量之间是否冗余；basis 是无冗余且能张成空间的向量组；dimension 则由 basis 中向量个数定义。

核心概念速查

概念 / Term解释
Linear independence如果只有所有系数都为 0 才能线性组合得到 zero vector，则向量组线性无关。
Span一个向量组所有 linear combinations 构成的集合。
Basis既 linearly independent 又 span 整个 space 的向量组。

定理讲解

Theorem: 在有限维 vector space \(V\) 中，任意两个 basis 的元素个数相同。这个结论使得 dimension \(\dim(V)\) 的定义是良定的，因为它不依赖于你选择哪一组 basis。

条件：空间必须是 finite-dimensional，并且讨论的是同一个 vector space 的 basis。结论：所有 basis 的 cardinality 相同。意义：dimension 可以作为空间本身的属性。

证明思路

证明通常依赖 replacement theorem。直觉是：如果一组 basis 比另一组长，那么较长那组中必然存在冗余向量，这会和 linear independence 矛盾。因此两个 basis 的长度必须相同。
